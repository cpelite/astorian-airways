Application Programming Interface

Unsere API erlaubt befreundeten Airlines, staatlichen Behörden und jedermann einen maschinenlesbaren Zugriff auf unseren Datenbestand. Um eine komplexe simulierte Umgebung zu erschaffen ist Informationsaustausch unerlässlich. Dies ist unser Beitrag hierzu. Sämtliche Schnittstellen sind ausschließlich lesbar und folgen dem RESTful Ansatz. Als Austauschformat wird JSON genutzt.

Airports

Alle Flughäfen in unseren System sind über eine RESTful API abrufbar. So sind alle ankommenden und abgehenden Routen von und nach Astoria City sowie diverse Meta-Information zum Flughafen selbst auf {{ url_for('api_airport_details',code='OJI') }} einsehbar. Die dreistelligen Codes für einzelne Flughäfen können den jeweiligen staatlichen Behörden oder {{ url_for('api_airports') }} entnommen werden.

Planes

Diverse Informationen zu unseren Flugzeugen können ebenfalls über die API abgefragt werden. Zusätzlich zu den Daten über Passagiere, Besatzung und Hersteller finden sich auch Angaben über die aktuelle Situation sowie aktuelle Koordinaten der Maschine in der API {{ url_for('api_plane_details', registration='A-ARYA') }}. Eine allgemeine Sicht auf all unsere Maschinen ist unter {{ url_for('api_current_planes') }} einsehbar. Sie enthält auch die Registrierungsnummern aller AA-Flugzeuge um mehr Details abzurufen.

Flights

Auch all unsere Flüge sind über die API einsehbar. Auf {{ url_for('api_flight_details', flight_number='AA5003') }} zum Beispiel werden Information zum Flug AA 5003 angezeigt. Neben Ankunft- und Abflugzeiten werden dabei auch die Wochentage, an denen der Linienverkehr stattfindet, ausgegeben. Die Flugdauer wird dabei in Minuten angezeigt. Es existiert auch eine Live-Karte mit aktuell stattfindenden Flügen. Bitte beachtet dabei die HTTP Cache-Header um die Server nicht ständig unnötig mit der Generierung neuer Karten zu belasten.
